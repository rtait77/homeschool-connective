## Session 4 — Consulting Webhook Debugging

### Root Cause Found
- Vercel serverless functions block outbound SMTP (port 587) — confirmed platform restriction
- `nodemailer` `sendMail()` was throwing an uncaught exception, crashing the entire webhook before any email could be sent
- This caused: no email to customer, no email to Mel, no `consulting_customers` record created, dashboard/intake not loading

### Fixes Applied
- **`app/api/webhook/route.ts`** — removed `nodemailer` entirely; Mel notification switched from Titan SMTP to Sender.net API (HTTP, not SMTP — works on Vercel). Both email sends wrapped in `try/catch` so one failure can't crash the other
- **`app/consulting/success/page.tsx`** — switched from `getUser()` to `onAuthStateChange` pattern to fix auth race condition after Stripe redirect

### Still Not Working — Needs Investigation Next Session
- After the webhook fix was deployed, another test payment was made and emails STILL did not arrive
- Possible remaining causes:
  1. Sender.net transactional API may not be authorized to send FROM `support@homeschoolconnective.com` (domain verification issue)
  2. Sender.net account may not have transactional email enabled (separate from newsletter/broadcast)
  3. The `consulting_customers` table may not exist in Supabase yet — if the insert throws in a way that crashes before emails, emails never send
  4. Stripe webhook may not be firing at all (check Stripe dashboard → Developers → Webhooks → recent events)

### Recommended Next Session Checklist
1. Go to Stripe dashboard → Developers → Webhooks → click your webhook endpoint → check "Recent deliveries" — were there any delivery attempts after the test payment? What was the response code?
2. If webhook delivered: check Vercel logs filtered to `/api/webhook` for the exact error
3. Check Sender.net account → confirm transactional email is enabled (not just newsletters)
4. Check Supabase → confirm `consulting_customers` table exists with columns: `id`, `user_id`, `stripe_session_id`, `paid_at`, `ends_at`
5. Clean up test consulting purchases from Supabase before next live test
