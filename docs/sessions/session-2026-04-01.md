## Bugs Fixed

### Webhook crash on subscription events
**Root cause:** `new Date((subscription as any).current_period_end * 1000).toISOString()` — `current_period_end` is `undefined` in Stripe SDK v17 webhook payloads, producing `new Date(NaN)` → `RangeError: Invalid time value`. This crash happened before any profile update or email send, so nothing worked.

**Fix 1:** Safely accessed `current_period_end` (null if undefined).
**Fix 2:** Tried `invoice.period_end` via expanded latest_invoice — returned today's date (setup invoice quirk).
**Fix 3 (final):** Computed renewal date from `subscription.created + 1 interval` — reliable for new subscriptions. Results in correct May 1 / April 1 next year dates.

Files: `app/api/webhook/route.ts`

### `subscription.updated` handler same crash
Same `current_period_end` issue — same fix applied. Uses fresh `stripe.subscriptions.retrieve()` with invoice expand as fallback.

### Auth callback overwrote active subscription status
`app/auth/callback/route.ts` unconditionally set `subscription_status: 'trialing'` for any user who passed through it. A subscriber clicking the magic link in the welcome email would have their subscription overwritten.

**Fix:** Check existing `subscription_status` first. Only set trial fields if status is null (new user).

## Consulting Flow Polish

### Email Mel removed from dashboard
Removed "Email Mel" button (appeared in two places in consulting section), the modal, and all associated state/function code. Users contact Mel by replying to emails directly.

### Intake form confirmation at bottom
Added green "Form submitted" banner at the BOTTOM of the intake form page (below nav buttons), in addition to the one at the top. Users no longer need to scroll up to confirm submission.

### Email field removed from intake form
The `parentEmail` field was redundant — `app/api/consulting/submit-intake/route.ts` already uses `user.email` from the auth session for all emails. Removed from form UI, TypeScript interface, and initial state.

### Consulting welcome email — Mel's copy
Replaced placeholder italic text with Mel's actual copy:
> "Hi! I'm Mel, and I'm so glad you're here. I can't wait to dig into your family's story and put together recommendations that actually fit how you learn and live. Your first step is to fill out the intake form below — take your time, it saves as you go. Once I receive it, I'll be in touch within a few business days."

### Consulting signup — password starts visible
`showPassword` initial state changed from `false` to `true`. Field shows as plain text by default with a "Hide" button.

### Pricing page — hide consulting card for existing consulting customers
Added `hasConsulting` state to pricing page. If the logged-in user has a `consulting_customers` record, the consulting card is hidden and the grid collapses to a single centered card. Also added `pricingReady` state so the grid only renders AFTER the auth/consulting check completes — prevents the flash where both cards appear then consulting disappears.
